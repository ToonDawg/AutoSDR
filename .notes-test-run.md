## Environment / setup observations

- **Setup wizard not shown.** Workspace row already exists, so
  `GET /api/setup/required` returns `{setup_required: false}` and
  `App.tsx` `SetupGate` skips the wizard. Expected behaviour for a
  pre-provisioned workspace.
- **`.env` carries `GEMINI_API_KEY`.** That is what's actually feeding
  LiteLLM, not a key in `workspace.settings`. The UI never asked for a

  expects** (`connector`, `auto_reply_enabled`, `rehearsal`,
  `llm.provider_api_keys`). Now self-healed on boot — see "Fixes"
  below.

## Fixes applied this run

### 1. HITL reason labels (frontend) — high visibility bug
- **Problem:** `frontend/src/lib/format.ts::HITL_LABEL` +
  `HitlReason` enum used keys (`eval_failed_max_attempts`,
  `connector_failed`, `reply_bot_check`, …) that did NOT match the
  backend-emitted strings (`eval_failed_after_3_attempts`,
  `connector_send_failed`, `bot_check`, …). Every HITL thread showed
  the raw enum string as its badge.
- **Fix:** Rewrote `HitlReason` + `HITL_LABEL` to mirror the exact
  strings produced by `autosdr/pipeline/{outreach,reply}.py` and
  `autosdr/api/threads.py`. Added labels for `reply_eval_failed`,
  `escalated`, and `taken_over_by_human` which were missing entirely.
- **Verified:** Inbox + Dashboard now render "Draft failed evaluator
  three times" on the real live threads.

### 2. Inbox `HitlRow` empty state — usability
- **Problem:** `eval_failed_after_3_attempts` and
  `connector_send_failed` contexts don't contain `suggestions` or
  `intent`, so the row rendered title + timestamp with no body. Could
  not triage without clicking into each thread.
- **Fix:** `HitlRow` now falls back to `hitl_context.last_drafts` +
  `last_scores` when `suggestions[0]` isn't available. Renders a
  "last attempt · NN" badge with the most recent draft text.
- **Verified:** Both live HITL rows now preview their last draft.

### 3. Landline re-import status reconciliation — production-risk bug
- **Problem:** `autosdr/importer.py::_process_row` refined
  `contact_type` on re-import but never updated `status` /
  `skip_reason`. A lead initially classified MOBILE (or UNKNOWN) that
  turned out to be a LANDLINE/TOLL_FREE stayed at `status=new` and was
  assignable to campaigns. In production with a real SMS connector
  this would have texted landlines. Current DB shows the fallout:
  28 landlines, 22 with `status=contacted` and 0 with `status=skipped`.
- **Fix:** When `contact_type` changes on re-import AND the lead is
  still in a pre-engagement state (`new` or `skipped`), reconcile:
  - non-mobile → `status=skipped` with
    `skip_reason="not_a_mobile_number:<type>"`
  - mobile → promote back to `status=new`, clear `skip_reason`
  Contacted/replied/won/lost leads are never touched — preserves
  history and avoids mass-demotion of already-engaged records.
- **Tests:** Added three regression tests to
  `tests/test_importer.py` (demote, promote, never-touch-contacted).
  Full importer suite: 15 passed.

### 4. Workspace settings self-heal on boot — legacy-workspace robustness
- **Problem:** Legacy workspaces (created by the old `init` CLI or
  pre-schema wizard) had settings blobs missing `connector`,
  `auto_reply_enabled`, `rehearsal`, and `llm.provider_api_keys`. The
  Settings page and various status defaults silently papered over
  these with `.get(..., default)` but the UI couldn't edit what wasn't
  there.
- **Fix:** `autosdr/webhook.py::_load_workspace_settings` now deep-merges
  `DEFAULT_WORKSPACE_SETTINGS` into any existing blob and persists the
  backfill once at lifespan startup, guarded so it's a no-op on an
  already-complete blob.
- **Verified:** `GET /api/workspace` now returns the full shape;
  Settings page shows populated connector/LLM/rehearsal sections.

### 5. Campaigns row layout — user-visible overlap
- **Problem:** "REPLIES" label overlapped "WON" column at the narrow
  `md` breakpoint because stat cells were ~42px but the label rendered
  at ~54px.
- **Fix:** Gave stats the wider slot (`md:col-span-5`), narrowed the
  name slot (`md:col-span-4`), added `truncate` + `title={label}`
  tooltip to `StatMini`, tightened `gap-x-3`, and added
  `whitespace-nowrap` to the QuotaMeter label so "24H SENT 21 / 50"
  stays on one line inside its column.
- **Verified:** Live Campaigns page renders cleanly; REPL… truncates
  with an ellipsis (+hover tooltip) instead of bleeding into WON.

## Follow-ups / backlog (not actioned this run)

- **LLM-calls table header overlap at narrow widths.** Same class of
  layout bug — `TOKENS/LATENCY/ATTEMPT/THREAD` columns collide. Fix is
  likely a combination of `min-w-0` + `truncate` + allowing horizontal
  scroll on the table container.
- **One-shot data migration for stale landlines.** The importer fix
  prevents future contamination, but the current DB still has
  28 landlines (22 `contacted`, 6 in misc states). Suggested one-off:
  flip every `Lead` whose `contact_type` is LANDLINE/TOLL_FREE *and*
  whose `status == new` to `skipped` with a proper `skip_reason`;
  leave contacted/replied/won/lost rows alone (they're historical).
- **Sidebar "current" highlight on `/threads/:id`** goes to "Threads"
  even when you got there from "Needs your eye". Cosmetic.
- **Setup wizard does not surface `GEMINI_API_KEY` from `.env`.** If
  the operator reads their key out of the env, the Settings LLM field
  stays empty. Consider either (a) hydrating the field from
  `os.environ` when the stored value is null, or (b) explicit copy in
  the Settings panel telling them it's read from env.
